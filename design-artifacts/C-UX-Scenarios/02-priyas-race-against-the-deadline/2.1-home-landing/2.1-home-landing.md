## Overview

**Page Purpose:** Get Priya to a working search within seconds of arrival — no detour through marketing content or browsing.

**User Situation:** Googled the exact textbook title while on campus, clicked directly into bookswap, phone in hand between classes.

**Success Criteria:** Priya enters her search query within the first few seconds on the page — the search bar must be found instantly, not discovered.

**Entry Points:**
- Google search result for the exact title + "used"/"swap" (external, primary — this is a deep-link-style visit, not a casual browse)
- Direct URL / bookmark (returning users)

**Exit Points:**
- Search Results (2.2)

## Layout Structure

Search-first hierarchy — the search bar is the largest, highest-contrast element on the page, above any hero/marketing content.

```
+----------------------------+
| bookswap logo              |
+----------------------------+
| [ Search title or ISBN  ] 🔍|
+----------------------------+
| (below the fold, optional) |
| Browse by genre →          |
| How it works →             |
+----------------------------+
```

---

## Page Sections

### Section: Search Bar
**Purpose:** The dominant, immediately-actionable element — this page exists to get Priya searching, not reading.
**Elements:**
- Text input with placeholder "Search title or ISBN"
- Auto-focus on page load (mobile keyboard opens immediately, no extra tap)

### Section: Secondary Content (below fold)
**Purpose:** Serves other entry patterns (Maya's browsing, first-time visitors) without competing with search for Priya.
**Elements:** Browse-by-genre link, brief "how it works" — deliberately below the fold, not competing with the search bar for attention.

## Technical Notes

- Auto-focusing the search input on load is a deliberate, persona-driven choice — validate it doesn't hurt Maya's browsing entry (she can still scroll past to Browse; the search bar being focused doesn't block scrolling).
