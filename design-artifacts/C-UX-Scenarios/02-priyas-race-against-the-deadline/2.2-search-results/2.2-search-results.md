## Overview

**Page Purpose:** Give Priya an unambiguous, immediate answer: is this exact title available or not. This page directly answers the highest-scored *want* in the whole Trigger Map (14/15).

**User Situation:** Just searched the exact textbook title, waiting for a yes/no.

**Success Criteria:** Priya knows within seconds whether a match exists — and if not, is offered an immediate, low-effort path to certainty (notify-me) rather than being left to wonder.

**Entry Points:**
- Home/Landing (2.1) — searches exact title

**Exit Points:**
- Notifications (2.3) — no exact match, activates alert (this scenario's sunshine path)
- (Out of scope for this scenario, but a real path) Book Listing Detail → Trade/Offer Flow, if a match IS found — see Scenario 03/04 for that continuation

## Page Sections

### Section: Match Status
**Purpose:** State clearly and immediately whether the exact title is available — no ambiguous "browse these similar results" framing when there's no exact match.
**Content:** Explicit no-match state (not a blank/empty list), with the searched title echoed back for confidence it searched correctly.

### Section: Notify Me CTA
**Purpose:** Convert "no match" from a dead end into instant certainty — the core promise this page makes to Priya.
**Elements:** Single prominent button, no additional form fields (uses her existing account, no re-entering title/email).

### Section: Matches List (conditional)
**Purpose:** If exact or close matches exist, show them — same visual pattern as Browse Listings for consistency, using Book Listing Detail as the template page.

## Page States

|-------|------|------------|---------|

## Technical Notes

- "No match" must be a distinct, deliberate state — not an empty-results list that could read as a bug or a failed search.
