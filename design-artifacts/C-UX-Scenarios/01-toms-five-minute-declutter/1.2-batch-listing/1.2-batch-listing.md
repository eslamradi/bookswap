## Overview

**Page Purpose:** Let Tom list every book in his box in one continuous sitting, with the minimum possible taps per book. This page IS the answer to the single highest-scored driving force in the entire Trigger Map (fear: listing effort outweighs payoff, 15/15).

**User Situation:** Just authenticated, box of books next to him, ready to go through them one by one.

**Success Criteria:** Tom lists all 8 books and reaches My Listings/Dashboard in under 8 minutes total (median target: under 2 min for the whole scenario per Goal 3.1 — batch listing is the page that has to deliver this).

**Entry Points:**
- Sign Up/Login (1.1) — only entry, first-time flow
- "Add more books" from My Listings/Dashboard (1.3) — return entry for adding another batch later

**Exit Points:**
- My Listings/Dashboard (1.3) — after bulk-submitting

## Layout Structure

A repeatable "card" pattern — one card per book, added to a running list as Tom scans/photographs each one. No per-book "submit" — one bulk action at the end.

```
+----------------------------+
| < 3 books added            |
+----------------------------+
| [ Scan next book (camera) ]|
+----------------------------+
| Book 1: [thumb] Title  ✓   |
|   Condition: [Good v]      |
| Book 2: [thumb] Title  ✓   |
|   Condition: [Fair v]      |
| Book 3: [thumb] Title  ✓   |
|   Condition: [Good v]      |
+----------------------------+
| [ List all 3 books ]       |
+----------------------------+
```

---

## Page Sections

### Section: Scan Entry
**Purpose:** Start each book with a barcode/ISBN scan, auto-filling title, author, and cover image — Tom never types a title.
**Elements:**
- Camera viewfinder (opens directly, no extra tap to "start scanning")
- Manual ISBN entry fallback (small link, for damaged/missing barcodes)

### Section: Book Queue
**Purpose:** Running list of books added this session, each requiring only one more decision.
**Elements per book card:**
- Auto-filled thumbnail, title, author (from ISBN scan)
- One photo capture (condition photo — single tap, camera opens inline)
- Condition preset selector: Good / Fair / Worn (single tap, no free text)
- No description field, no pricing field, no "exchange or sell" choice per book — those can be bulk-set defaults or deferred to Dashboard editing, kept OUT of this page to protect speed

### Section: Bulk Submit
**Purpose:** One action lists everything in the queue — reinforces "fast, almost no effort."
**Elements:**
- "List all N books" button, always visible (sticky footer), count updates live

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page load, first book | Camera viewfinder prominent, empty queue | Scan a book |
| Scanning | Mid-scan | Brief recognition animation | None — auto-fills on success |
| Scan failed | Barcode unreadable | Inline prompt: "Couldn't read that — try again or enter ISBN manually" | Retry or manual entry |
| Queue building | 1+ books added | Queue visible below scanner, submit button shows count | Scan another, edit a card, or submit |
| Submitting | Bulk submit tapped | Brief loading state on submit button | None — auto-advances to Dashboard |
| Submit error | Network/server failure | Queue preserved, inline error, retry available | Retry submit — queue is NOT lost |

---

## Technical Notes

- Queue state must persist locally (survive accidental navigation/app backgrounding) — losing Tom's in-progress queue is the single worst thing that can happen on this page given his abandonment risk.
- ISBN scan → auto-fill is the core speed mechanism; a book catalog/metadata lookup service is a hard dependency for this page to hit its friction targets.
