## Overview

**Page Purpose:** Let a viewer evaluate condition and trust signals closely enough to commit to requesting the book — this page carries the highest-scored fear in the Trigger Map's condition-trust cluster (13/15, Maya's fear: condition worse than described).

**User Situation:** Selected a listing from Browse Listings (Maya) or Search Results (Priya) — arrives wanting to decide "is this trustworthy enough to request."

**Success Criteria:** Viewer trusts the condition description enough to proceed to requesting/checking the lister's profile.

**Entry Points:**
- Browse Listings (3.1) — Maya's discovery path
- Search Results (2.2) — Priya's match path

**Exit Points:**
- User Profile/Reputation (3.3) — verify lister trust before committing (Maya's path)
- Trade/Offer Flow (4.1) — direct request path for users who already trust the platform (Priya's likely path, out of scope for this scenario's documentation but same page)

## Page Sections

### Section: Photo Evidence
**Purpose:** Directly answer "is the condition as described" — this is the single highest-leverage element on the page for trust.
**Elements:** Multi-photo carousel (required at listing time, not optional), zoomable.

### Section: Condition & Details
**Purpose:** Structured, not free-text, condition rating (reuses the same Good/Fair/Worn preset from Batch Listing for consistency), plus title/author/edition metadata.

### Section: Description
**Purpose:** Lister's own notes — free text, secondary to the structured condition rating (structured data is trusted more than prose).

### Section: Lister Trust Summary
**Purpose:** Compact trust signal without requiring a full profile visit — rating badge + name + avatar, links to full profile (3.3).

### Section: Primary Action
**Purpose:** Single clear CTA to request the book — the page's conversion point.
