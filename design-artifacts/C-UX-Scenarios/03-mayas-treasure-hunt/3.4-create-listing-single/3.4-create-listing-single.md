## Overview

**Page Purpose:** Let Maya list one of her own books to offer in trade — the scenario's success point, completing the swap request.

**User Situation:** Just confirmed the other lister is trustworthy, ready to offer something in exchange.

**Success Criteria:** Maya sends a trade request backed by one of her own books, in a single, unhurried flow (unlike Tom's batch flow, this is a single, more considered listing — Maya isn't optimizing for speed the way Tom is).

**Entry Points:**
- User Profile/Reputation (3.3) — after confirming trust

**Exit Points:**
- None required — scenario success. Trade request sent, Maya waits for response.

## Layout Structure

Reuses the same core listing pattern as Batch Listing (scan → auto-fill → condition → photo), presented as a single-item flow rather than a repeatable queue.

```
+----------------------------+
| Offering in trade for:      |
| [thumb] "Their Book Title" |
+----------------------------+
| [ Scan your book (camera) ]|
+----------------------------+
| [thumb] Your Book Title    |
|   Condition: [Good v]      |
+----------------------------+
| [ Send trade request ]     |
+----------------------------+
```

---

## Page Sections

### Section: Context Reminder
**Purpose:** Keep Maya oriented — shows what she's trading FOR, so the single-listing flow doesn't feel disconnected from her original goal.

### Section: Book Entry
**Purpose:** Same scan → auto-fill → condition → photo pattern from Batch Listing, reused for consistency, presented once (not a queue).

### Section: Send Request
**Purpose:** Single clear action that completes the scenario.

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
