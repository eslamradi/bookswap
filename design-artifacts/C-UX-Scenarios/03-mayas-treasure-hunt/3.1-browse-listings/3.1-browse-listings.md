## Overview

**Page Purpose:** Let Maya discover an unexpected book through unhurried, pleasurable browsing — the deliberate opposite of Priya's task-driven search.

**User Situation:** Free evening, just finished her last book, opens bookswap out of habit, settled on the couch with a tablet.

**Success Criteria:** Maya finds something she wouldn't have searched for by name and taps into it — the "treasure hunt" feeling is the success metric here, not raw conversion speed.

**Entry Points:**
- Home nav / habitual direct visit (primary — repeat, engaged user)
- "Browse by genre" link from Home/Landing (2.1) — secondary path for less frequent visitors

**Exit Points:**
- Book Listing Detail (3.2) — on selecting a listing

## Layout Structure

Grid-based, visual-forward — book covers do the work, not dense text. Designed for scrolling and lingering, not scanning-and-leaving.

```
+----------------------------+
| Filters: Genre  Condition  |
+----------------------------+
| [cover] [cover] [cover]    |
| [cover] [cover] [cover]    |
| [cover] [cover] [cover]    |
|  ... (infinite scroll)     |
+----------------------------+
```

---

## Page Sections

### Section: Filters
**Purpose:** Let Maya narrow to genres she likes without forcing a rigid search — filters, not search.
**Elements:** Genre chips (multi-select), condition filter, "open to any" vs. "specific listings" toggle if relevant.

### Section: Listing Grid
**Purpose:** Visual, cover-forward browsing surface designed for scrolling.
**Elements:** Cover thumbnail, title, author, condition badge, lister trust indicator (small rating badge) — enough to decide "worth a closer look" without a full page load per item.

---

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page load | Grid populated, no filters active | Scroll, filter, or select a listing |
| Filtered | Genre/condition filter applied | Grid updates to match | Adjust filters further, or select |
| Empty (post-filter) | No listings match active filters | Friendly empty state, suggests loosening filters | Clear filters |
| Loading more | Infinite scroll fetch | Skeleton cards at bottom of grid | None |

---

## Technical Notes

- MVP version: reverse-chronological / filterable grid is sufficient (per Feature Impact Analysis — basic discoverability ships with MVP). A personalized recommendation algorithm ("someone like you also swapped") is explicitly a post-MVP investment once there's usage data to power it — do not block launch on this.
