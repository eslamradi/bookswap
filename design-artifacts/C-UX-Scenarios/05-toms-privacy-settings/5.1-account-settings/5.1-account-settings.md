## Overview

**Page Purpose:** Let Tom set address-masking / platform-mediated shipping as his default before he ever lists a book — proactively resolving his top privacy fear (13/15) before it's forced on him mid-trade.

**User Situation:** Just finished signing up, hasn't listed anything yet, privacy is on his mind before it becomes an issue.

**Success Criteria:** Default is set in under a minute; Tom never has to think about it again mid-trade (directly removes the decision from Shipping/Handoff Coordination, 4.4, later).

**Entry Points:**
- Profile menu, right after sign-up

**Exit Points:**
- None required — scenario success. Default is set; Tom moves on to listing.

## Page Sections

### Section: Privacy Toggle
**Purpose:** Single, clear on/off control for address-masking, defaulted ON — Tom shouldn't have to discover this exists, it should already be protecting him.

### Section: Default Shipping Method
**Purpose:** Pre-select the same choice offered later in Shipping/Handoff Coordination (4.4), so that page can skip the question entirely for Tom next time.

### Section: Reassurance
**Purpose:** Explicit statement of the guarantee, same pattern as used on 4.4 — consistency reinforces trust.

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page load | Address masking ON by default, no shipping method pre-selected | Confirm/change shipping method preference |
| Saved | After any change | Brief inline confirmation | None — auto-saves |

---

## Technical Notes

- Address masking should default to ON for all new users, not just be available as an opt-in — this page is about giving Tom visibility/control over a protection that should already exist, not asking him to turn on safety.
