## Purpose

Let Tom queue up multiple books in one sitting with minimal per-book effort, then bulk-submit. This page answers the single highest-scored driving force in the Trigger Map (fear: listing effort outweighs payoff, 15/15).

## Prototype Limitation

No real camera/barcode scanning available in a static browser prototype. Scanning is simulated via the spec's own manual-ISBN fallback: selecting/typing one of 8 demo ISBNs returns the matching book instantly (same UX outcome a real scanner would produce — auto-filled title/author/cover). Photo capture is simulated via a toggle rather than an actual camera.

## Objects & Behavior

| Object | Object ID | Behavior |
|--------|-----------|----------|
| Live count header | `batch-listing-count` | Updates as queue grows |
| ISBN input (simulated scan) | `batch-listing-isbn-input` | Enter/select ISBN → lookup |
| Scan button | `batch-listing-scan-btn` | Triggers lookup |
| Scan-failed message | `batch-listing-scan-error` | Shown on unrecognized ISBN |
| Queue container | `batch-listing-queue` | Holds book cards |
| Book card (per item) | `batch-listing-card-{id}` | Thumbnail, title, author |
| Condition selector (per card) | `batch-listing-condition-{id}` | Good/Fair/Worn preset |
| Photo toggle (per card) | `batch-listing-photo-{id}` | Simulated capture |
| Remove button (per card) | `batch-listing-remove-{id}` | Removes from queue |
| Bulk submit button | `batch-listing-submit-btn` | Submits entire queue |
| Submitting state | (button state) | Spinner, disabled |
| Submit-error banner | `batch-listing-submit-error` | Queue preserved on failure |

## Auth Guard

Page checks `PrototypeAPI.getUser().authenticated` on load; redirects to `1.1-sign-up-login.html` if not authenticated (consistent with spec entry point: only reachable after sign-up).

## Persistence

Queue is stored via `PrototypeAPI.addToQueue/updateQueueItem/removeFromQueue`, backed by localStorage — survives refresh/navigation per the spec's technical requirement ("losing Tom's in-progress queue is the single worst thing that can happen on this page").
