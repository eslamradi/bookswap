## JavaScript Requirements

- `handleAuth(method)`: disables all auth buttons, shows spinner on the clicked button, calls `window.PrototypeAPI.authenticate(method)`, on success navigates to `1.2-batch-listing.html`, on failure shows `signup-login-error` and re-enables buttons.
- Since `PrototypeAPI.authenticate()` is a local mock (localStorage), it always succeeds — the error path is included per spec but can only be exercised by simulating a failure (documented as a known limitation of a static prototype, not tested live).

---

## Demo Data

Uses `window.PrototypeAPI.authenticate(method)` from `shared/prototype-api.js`, which sets `currentUser.authenticated = true` in localStorage (seeded from `data/demo-data.json`).
